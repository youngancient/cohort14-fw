# 🤝 Contributing to MultiSig Project- Group E

This project aims to build a secure, scalable MultiSig system with a clean frontend and robust smart contracts.

---

## 📌 Contribution Guidelines

We welcome contributions in the following areas:

* 🧠 Smart contract development (Solidity / Foundry / Hardhat)
* 🎨 Frontend improvements (React, TypeScript, UI/UX)


---

## 🛠️ Getting Started

### 1. Fork the Repository

Click the **Fork** button on GitHub and clone your fork:

```bash
git clone https://github.com/OperaCode/multisigFR.git
cd REPO_NAME
```

---

### 2. Install Dependencies

#### Frontend:

```bash
cd frontend
npm install
```


### 3. Create a Branch

Always create a new branch for your work, use your name to create the branch:

```bash
git checkout -b feature/your-name
```

Examples:

* `feature/samuel`
* `fix/isaac`
* `refactor/zeus`

---

## 🧑‍💻 Development Workflow

### 💡 Code Style

* Use **TypeScript** for frontend
* Follow **ESLint + Prettier** rules
* Keep components modular and reusable
* Use clear and descriptive naming

---



## 📦 Commit Guidelines

Use clear and meaningful commit messages:

```bash
feat: add multisig transaction approval flow
fix: resolve nonce replay vulnerability
refactor: simplify execution logic
test: add edge case tests for withdrawals
```

---

## 🚀 Pull Request Process

1. Push your branch:

```bash
git push origin feature/your-feature-name
```

2. Open a Pull Request (PR)

3. Ensure:

* ✅ Code compiles
* ✅ Tests pass


4. Add a clear PR description:

* What you changed
* Why it was needed
* Screenshots (if UI-related)
